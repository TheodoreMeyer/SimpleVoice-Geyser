# Contributing Guidelines

### Rules
- Contributed Code must be tested before being merged into Main Branch. This means that Pull requests should be opened first.
- Any Code contributed May NOT contain any viruses, Malware, or any other code of that type.
- Everything in contributed code MUST follow the [Minecraft EULA](https://www.minecraft.net/en-us/eula).

### First Time Contributors
- Please make sure that a Maintainer reviews your code.

### Pull Requests
- Please make sure that your Pull Request is well described, and that you have tested the code
- Fill out the PR template, and make sure to check the boxes that apply to your PR.
- Please make sure that your PR is not a duplicate of another PR, if you have the same change as another PR, please comment on the other PR your change.

### Showing Contributions
- If you want to be recognized for your contribution, please make sure to add your name to the Contributors.md file, and make sure to add a link to your GitHub profile.
     - For example: `- [TheodoreMeyer](https://github.com/theodoremeyer): Role in the project`
     - Make this part of your PR.

### Issues
- Please make sure that All issues are well described, and are not closed unless:
  - the problem is fixed, as said the issue owner.
  - we think the problem is fixed, and the owner has not responded (but then a maintainer has to be the one to close it).
- Issues can't be duplicates of others, if you have the same problem, please comment on the other issue your problem.
